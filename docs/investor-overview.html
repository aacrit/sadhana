1. Executive Summary

Sadhana is the world's first browser-based Hindustani classical music training application built on a comprehensive music physics engine. It transforms the student's device into an interactive instrument that listens, responds, and teaches — all at $0 operational cost.

The core insight: Existing music apps teach Western music theory or treat Indian music as a subset of Western frameworks. Sadhana builds from first principles: 22 shrutis (microtonal positions), just intonation frequency ratios, raga grammar rules, and the oral-tradition pedagogy of Hindustani classical music.

2. The Problem

Market Gap

Hindustani classical music has 300+ million practitioners across South Asia and the diaspora. Yet:

  • No digital tool teaches raga grammar (melodic movement rules, not just scales)
  • No app uses just intonation — the mathematically correct tuning system that defines Indian music
  • No platform provides context-aware pitch feedback that understands the difference between komal Re in Raga Bhairav vs. komal Re in Raga Kafi
  • Western music apps (Yousician, Simply Piano) serve 50M+ users but are structurally incapable of teaching Indian music correctly

The Opportunity

  • Global Indian diaspora: 32 million people, many seeking cultural connection through music
  • Growing mindfulness market: Hindustani music is inherently meditative (riyaz = disciplined daily practice)
  • Underserved market: No serious competitor exists in this space

Tantri: The Instrument Layer

Tantri ("string of a veena") is a 12-string interactive instrument rendered on HTML5 Canvas. It is simultaneously input (voice maps to string vibrations) and output (touch triggers synthesis).

  • Voice input: Each string vibrates when the student's voice is near that swara's frequency. Color encodes accuracy: saffron = perfect, green = good, amber = approaching.
  • Sympathetic vibration: Pa vibrates when Sa is sung, matching real tanpura physics.
  • Spring physics: Kan snap (stiffness 1000), Andolan sustain (120/8), Tanpura Release (400/15, ~800ms decay).
  • Progressive disclosure: Shishya sees 1 string (Sa). Raga strings emerge. The instrument grows as the student grows.
  • Keyboard accessible: ArrowUp/Down navigation, Enter/Space to play.

5. User Lifecycle: Onboarding to Mastery

Onboarding (2 minutes)

  1. Sa Detection: Student hums a comfortable note 3–5 times. Engine proposes "Your Sa is G3 (196 Hz)." Stored permanently.
  2. Journey Selection: Four paths visible immediately. All show the full vision. No gates on visibility.

Daily Session Loop

  1. On open: Today's riyaz prepared. Raga chosen by time of day (Dawn → Bhairav, Afternoon → Bhimpalasi, Dusk → Bhoopali, Evening → Yaman, Night → Bageshri).
  2. Tanpura begins. Student sings. ~10 minutes. Zero decisions required.
  3. On completion: Transitions to free practice. No friction between ritual and freedom.
  4. Daily goal ring on home page shows completion status.

Four Journey Paths

Shishya (Beginner)

8 lessons • Sa detection, 5 core ragas, guided riyaz, voice feedback

Sadhaka (Explorer)

10 lessons • 10 more ragas, ornaments, ear/interval training, meend, gamak

Varistha (Scholar)

11 lessons • Complex ragas, shruti discrimination, composition, tala integration

Guru (Master)

10 lessons • Advanced raga identification, bandish, taan, teaching tools

Progression System

  • Levels unlocked by musical acts, not XP. Example: "Sing Bhairav's pakad within ±20 cents across 3 separate sessions" → Sadhaka.
  • No popups, no confetti. Interface deepens invisibly. Student notices one day — that's the ceremony.
  • XP tracks consistency only (streak maintenance, sessions completed). It is the practice log, not the grade.

Pakad Recognition Moment

When the student sings the characteristic phrase of their current raga, the engine recognizes it in real time. A cinematic pause: tanpura continues, background deepens, raga name appears in Cormorant Garamond, the phrase in sargam notation fades in. This bridges Beginner and Scholar in one unrepeatable moment.

6. Curriculum & Raga Catalog

Pedagogical Principles

  1. Audio Before Everything: Every concept is heard before it is named.
  2. The Presence Rule: Tanpura plays for 30 seconds before any instruction appears.
  3. Raga-first: Students learn Bhoopali (a raga), not "the pentatonic scale."
  4. Progressive disclosure: Complexity emerges from musical experience.

9. Data Model

profiles              -- user identity, Sa reference (Hz), level, XP
  |
  +-- sessions        -- practice sessions: raga, duration, accuracy, XP
  |
  +-- streaks         -- current streak, longest streak, last riyaz date
  |
  +-- raga_encounters -- per-raga: session count, minutes, best accuracy, pakad found

Row-Level Security (RLS) policies ensure users only access their own data. All tables indexed for query performance. The 90-day practice heatmap aggregates from the sessions table.
